limitations of manual testing 

1. Re-testing - do the same test case same steps multiple time with different data. challenge: spend a lot of time and effort.
2. Regression testing - 
    1. developer build software as group of features build togethers. when the first build is done. as a testers test some test cases some of them may failed so make bug reports to developer. then the next build they fix the bugs and build new features. as a tester verify the fixed bug and the some dependent functionality. 
    2. to make sure the existing functionality should not be broken because of the changes done by developer.
    3. done in every cycle 

cycle 1:  TC [ 8 pass, 2 failed] — the 8 is automated 

cycle 2:  the  2 TC failed, tested manually — then the 8 TC again executed because the change happen. if 2 TC pass then automate them.
